Protocolo HTTP

  • Hypertext Transfer Protocol (Protocolo de Transferência de Hipertexto)
  • Protocolo que lida com o modelo de requisição/resposta utilizado para comunicação na Web
  • Executado sobre o protocolo TCP/IP

Modelo Resquest/Response: Usuário envia uma requisição, e recebe uma resposta.

Protocolo HTTP: Características

Stateless: Requisições Independentes (Não mantém "histórico" de execução)

Comunicação: Segue modelo Request/Response ("troca de mensagens" entre cliente e servidor)

  • Request: Requisição envada pelo usuário para o servidor
  • Response: Resposta do servidor para o usuário

HTTP Request

Responsável por enviar a requisição ao servidor

  • Composto por três parâmetros principais:
    • A página a ser acessada (url)
    • Parâmetros do usuário
    • Método HTTP (a ação a ser executada)

Métodos HTTP

Define a ação que será executada no servidor no momento do recebimento da requisição

  • GET: Leitura de informação
  • POST: Armazenamento de informação
  • PUT: Atualização de informação
  • DELETE: Exclusão de informação

Métodos HTTP: GET

  • Utilizado para operações de leitura, recuperação de informação
  • Pode-se enviar dados através do GET, porém não é aconselhável:
    1. Quantidade de caracteres é limitada
    2. Os dados são enviados via URL, tornando as informações públicas

Métodos HTTP: GET

Métodos HTTP: POST

  • Diferentemente do GET, possui recursos para transferência segura de informações
  • As informações são enviadas no corpo da requisição
  • Utilizado para persistência de informações

Métodos HTTP: POST

HTTP: Response

Resposta proveniente do servidor

Contém cabeçalho com informações gerais da resposta

Corpo compreende o conteúdo da resposta em si

HTTP: Response

  • Composto por três parâmetros principais:
    • Status (request foi um sucesso?)
    • Content-type (text, picture, HTML, etc.)
    • O conteúdo (HTML, imagens, vídeo, etc)

O Problema das Páginas Estáticas

  • Um servidor web lida apenas com conteúdo estático
  • Isto é, ele apenas retorna recursos já existentes no servidor
  • Todo cliente recupera o mesmo recurso

Aplicações Web

O servidor Web pode delegar essas ações para outra aplicação

Essa aplicação auxiliar é conhecida como Servidor de Aplicação Web

Aplicações Web: Vantagens

Possibilidade de processar informações

Capacidade de gerar páginas dinâmicas

Funcionamento transparente para o usuário (não sabe o que a aplicação realmente processa)
